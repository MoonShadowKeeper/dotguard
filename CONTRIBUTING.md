# Contributing to dotguard

Welcome! We appreciate your interest in making `dotguard` better. 

## Development
1. Fork the repo and clone it locally
2. Create a feature branch
3. Run tests with `node test/run.js` before submitting a PR
4. No external npm dependencies are allowed. This is a strict constraint to ensure security and speed.

## Pull Requests
- Keep PRs focused on a single issue or feature.
- Include an update to the tests if you are adding new functionality.
- Write clear commit messages.
